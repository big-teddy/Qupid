## ✅ UI 변경 시 필수 절차

### 1. 변경 전
```bash
# 원본 저장소 확인
git remote -v
# 관련 원본이 있는지 사용자에게 질문

# 현재 상태 스크린샷 저장
# 브라우저에서 직접 확인 후 캡처
```

### 2. 변경 중
- **한 번에 하나의 컴포넌트만 수정**
- 변경 후 즉시 브라우저에서 시각적 확인
- HMR(Hot Module Reload)이 제대로 작동하는지 확인

### 3. 변경 후
- 전체 화면 흐름 테스트 (온보딩 전체, 로그인 등)
- 다양한 화면 크기에서 테스트
- 변경 전/후 비교
